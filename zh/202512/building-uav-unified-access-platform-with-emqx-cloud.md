## 引言

随着低空经济的快速发展，无人机在物流、巡检、农业等领域的应用日益普及。行业迫切需要构建统一的云端平台，以实现大规模无人机的远程监控、实时控制与数据分析，并满足全球接入、高并发和低时延的严苛要求。

为此，某企业基于 EMQX 构建了无人机统一接入平台。该平台实现了无人机运行状态与任务进度的实时采集，并支持远程控制与任务下发等核心功能，为各类应用场景提供可靠的技术支撑。

随着业务边界持续拓展，该平台凭借出色的扩展性与兼容性，有效支持了企业多产品线的设备接入与业务扩展。通过统一技术架构，该企业不仅降低了系统建设成本，还提升了平台的可维护性与演进能力，为业务创新与持续发展奠定了坚实基础。

## 项目挑战

在构建云端连接平台的过程中，企业主要面临以下六大核心挑战：

### **技术通用性**

工厂产品线丰富，不同设备及配套系统采用不同的数据格式。平台需要兼容多种数据类型，并能在云端将来自设备、应用和第三方系统的数据进行统一集成。

### **大规模高并发连接**

企业业务覆盖全球，高峰期时数十万设备同时在线，数据吞吐量极大。平台必须能够承载百万级以上的并发连接和数据吞吐，避免在业务高峰出现通信瓶颈。

### **实时稳定的双向通信**

无人机在飞行中需要实时上报位置、状态及传感器数据，同时云端平台也需要下发遥控指令和任务信息。平台必须确保毫秒级的低时延和高可靠的双向消息传输。

### **数据安全与系统可靠性**

平台需要支持安全通信与细粒度权限控制，并能够完整记录设备事件及上报数据，以满足安全审计要求。同时，平台还应具备高可用与容灾能力，确保关键业务的实时性与可靠性。

### **多实例统一运维**

为隔离不同产品线的业务逻辑，工程师需要为每条产品线创建独立实例并进行运维管理，运维复杂度显著上升。平台需提供统一的管理界面与全托管服务，降低多实例运维成本，提升运营效率。

### **全球部署与低延迟服务**

该企业产品面向全球用户，平台需要支持多个消费市场就近部署，以保证不同国家和地区的用户都能低延迟、高可用的接入服务。

## 解决方案

为应对上述挑战，该企业最终选择了 [EMQX Cloud](https://www.emqx.com/zh/cloud) 作为云端消息平台的核心基础设施，系统性地解决设备连接和数据集成的关键难题。

EMQX Cloud 提供了全托管的 MQTT 服务，可灵活适配多样化的业务场景，同时支持快速创建并统一管理多个业务实例，让开发团队能够专注于上层应用的开发与创新。

### 高效数据汇聚与远程控制

平台基于轻量级 MQTT 协议，通过 4G/5G 及有线网络，与遥控器、无人机场建立双向实时连接。

无人机飞行中产生的大量遥测与状态数据，通过 EMQX 规则引擎桥接至 Kafka，统一汇入企业数据平台，并进行实时流处理、轨迹回放、故障排查与审计分析，为智能调度提供数据支撑。此外，数据也可通过 MQTT 实时推送至 App 端供用户查看。

在远程控制场景中，用户基于无人机上报的实时数据下发控制指令，指令由 EMQX 转发至无人机侧遥控器，再透传到无人机飞控系统。依托 EMQX 的低延迟与高可靠特性，在远程实现无人机的实时精准控制，并确保用户操控体验的流畅稳定。

平台兼顾了响应速度与指令精度，即使在大规模机群并发作业时，仍能确保数据传输稳定、通信流畅无阻。

### 无人机机场与任务系统集成

平台接入了无人机场与调度系统，用户可在云端完成航线规划与任务管理，通过 EMQX HTTP API 将任务包下发至指定机场设备。无人机根据任务计划自动起飞、巡航和执行作业，实现远程调度与自动化运行。

在任务执行过程中，无人机将实时回传进度、飞行日志与任务数据，实现全程可视化监控。任务完成后，用户可查看完整作业报告并回放任务执行过程，确保任务可追溯、可验证。

通过「云 - 地」协同的集成方式，该企业形成了「云端调度 - 地面执行 - 数据回传」的闭环体系，真正实现了无人值守调度，显著提升商业场景下的运营效率与管理水平。

### 设备连接与通信安全保障

EMQX 提供端到端安全防护，所有 MQTT 通信均采用 TLS 加密，并通过 HTTP 认证机制验证设备身份，确保各类飞行业务的通信安全与数据可靠性。

在上下行消息的权限控制方面，平台基于 EMQX 内置的授权检查功能，结合 ACL 缓存降低访问延迟。同时，针对敏感主题专门开发 ACL 缓存排除机制，确保权限变更即时生效，兼顾性能与安全。

借助 EMQX 的高性能数据集成能力，所有运行数据与指令下发均通过 Kafka 集成至业务平台并统一存储，实现全链路可追踪、可验证，满足企业在全球范围内的安全与合规要求。

### 多区域、高可用部署

基于 EMQX Cloud 的全球部署能力，该企业在国内外多个区域部署了生产与测试实例，用户可以就近接入，通信延迟低、服务连续性高。

为保障无人机远程控制等关键业务的高可用性，EMQX Cloud 支持多可用区部署。同一地域内，可将同一集群分布在不同可用区内运行，即使某一可用区故障，连接也能自动切换，确保指令下发与数据传输的稳定可靠。

### 多集群业务隔离

针对不同产品线与业务在安全、性能与扩展方面的独立需求，该企业为各业务创建独立的 EMQX 实例，实现连接层隔离。该架构有效避免业务间干扰与资源冲突，便于根据业务特点按需调优与弹性扩展。
