我们很高兴地宣布，EMQX Cloud 迎来重要版本更新。本次更新聚焦于平台扩展与区域可用性，旨在为物联网（IoT）团队提供更灵活的全球化 MQTT 基础设施部署方案。

**本次更新主要包括两项内容：**

- EMQX Cloud 专有版现已支持 Oracle Cloud Infrastructure（OCI）。
- 新增 Google Cloud 加拿大（多伦多）区域，满足北美客户的数据驻留与合规需求。

这些更新进一步巩固了 EMQX Cloud 作为真正的多云、全球化 MQTT 数据平台的地位。

## **EMQX Cloud 现已支持 Oracle Cloud Infrastructure**

EMQX Cloud 正式支持在 **Oracle Cloud Infrastructure（OCI）**上进行部署。

专有版用户可以在 OCI 上直接部署完全托管的单租户 EMQX Broker，而无需承担自行管理基础设施的运维开销。

**首批开放区域：**

- **美国东部：**阿什本 `us-ashburn-1`
- **欧洲：**法兰克福 `eu-frankfurt-1`

## 为什么 OCI 的支持对 IoT 团队至关重要？

EMQX Cloud 是首个且唯一在 OCI 上提供全托管服务的独立 MQTT 数据平台。

**对于已采用 OCI 标准的物联网团队而言，这意味着：**

- MQTT 基础设施可直接部署在既有的 OCI 环境中，无需跨云迁移数据，契合企业云战略。
- 部署逻辑与 OCI 的网络、安全及合规标准原生对齐，降低集成复杂度。
- 获得具备企业级可靠性、可扩展性和可观测性的全托管 MQTT 服务。

OCI 在制造业、能源、电信及企业 IT 等行业中应用广泛，这些领域对系统性能的可预测性和合规性有着极高要求。通过支持 OCI，EMQX Cloud 为以 Oracle 为中心的物联网开发者提供了一个卓越的 MQTT 平台，在享受企业级服务的同时，无需承担额外的运维负担。

### **专为数据驻留与合规性设计**

新增的 GCP 加拿大区域可以帮助客户：

- 将数据保留在加拿大境内，满足数据驻留要求。
- 显著降低加拿大及美国北部用户的访问延迟，提升实时响应速度。
- 符合当地的合规性需求与监管政策。

对于在加拿大市场开展业务或服务于该市场的组织，本次更新为其在 Google Cloud 上运行 MQTT 工作负载提供了一个兼顾合规与低延迟的清晰选择。
