## 新增驱动：**SNMP**、**GE Historian** 和 **OPC AE**



### SNMP 驱动：网络设备监控的标准协议



**SNMP（Simple Network Management Protocol，简单网络管理协议）** 是网络设备监控和管理的事实标准。通过 SNMP 协议，您可以实时监控路由器、交换机、服务器、UPS 等网络设备的状态和性能指标。

**核心能力：**

- 支持 **SNMP v2c** 版本
- 监控网络设备的 CPU、内存、带宽、端口状态等关键指标。
- 获取设备运行时间、系统信息、接口流量等数据。
- 适用于数据中心、工厂网络、楼宇自动化等场景。

**应用场景：**

- **数据中心监控**：实时监控服务器集群的 CPU、内存、磁盘使用率等。
- **工厂网络管理**：监控工业交换机的端口状态、流量负载。
- **楼宇自动化**：实时监控 UPS 电源、空调系统的运行状态。
- **AI 数据中心建设监控**：随着 AI 大模型训练和推理需求的爆发式增长，AI 数据中心的建设规模不断扩大。SNMP 驱动可以实时监控 GPU 服务器集群、高速网络交换机（如 InfiniBand、RoCE）、智能 PDU（配电单元）、精密空调等关键基础设施的运行状态，确保 AI 算力平台的稳定性和高效运行。通过 NeuronEX 将 SNMP 数据统一接入 MQTT，可以实现跨数据中心的集中监控、智能告警和能耗优化。

### GE Historian 驱动：访问工业历史数据库



**GE Historian** 是通用电气（GE）推出的工业历史数据库，广泛应用于能源、化工、制造等行业，用于存储和检索长时间跨度的工业过程数据。通过 NeuronHUB 访问运行于 Windows 操作系统的 GE Historian 服务器

- 读取历史趋势数据、实时数据、统计数据。
- 支持聚合计算（结合数据处理模块）。
- 将历史数据与实时数据融合，支持高级分析场景（结合数据处理模块）。

### OPC AE 驱动：工业报警与事件管理



**OPC AE（OPC Alarms and Events，OPC 报警与事件）**是 OPC 基金会定义的报警和事件管理标准，专门用于获取设备的报警、事件和状态变化信息。

**核心能力：**

- 通过 **NeuronHUB** 访问运行于 Windows 操作系统的 OPC AE 服务器，实时接收报警信息。
- 支持三种事件类型：
  - **Simple（简单事件）**：基本的状态变化通知。
  - **Conditional（条件事件）**：带条件判断的报警（如温度超限）。
  - **Tracking（追踪事件）**：操作员动作追踪（如确认报警、添加注释）。

**应用场景：**

- **集中报警管理**：将多个 PLC 和 SCADA 系统的报警统一汇聚到云端。
- **报警分析**：统计报警频率、响应时间，优化报警策略。
- **合规审计**：记录操作员对报警的确认和处理过程。

## 数据采集管理全面升级



### 北向应用导入导出



新增 **北向应用配置的一键导入与导出** 功能，简化了多实例部署和配置迁移的工作量。

- **批量复制**：将一个项目的北向配置快速复制到其他项目。
- **配置备份**：定期导出配置文件，确保配置安全。
- **标准化部署**：在多个边缘节点间复用标准化的北向应用配置。

### 标签管理系统



南向驱动与北向应用现在均支持添加 **"标签"（Tag）**，并支持基于标签进行搜索和筛选。

- 为设备添加自定义标签（如 `生产线1`、`关键设备`、`测试环境`）
- 基于标签快速筛选和查找设备
- 支持多标签组合搜索

### 南向驱动跨页筛选



南向驱动列表页面现在支持**跨页筛选**功能：

- 根据「工作状态」（运行中/已停止）筛选。
- 根据「连接状态」（连接/断开）筛选。
- 筛选结果包含所有页面的匹配设备，不再局限于当前页。

## AI 深度集成：智能化规则编写



NeuronEX 3.8.0 将 AI 能力深度集成到数据处理流程中，显著降低了工业数据分析的技术门槛。

在创建数据处理规则时，AI 助手现已深度集成数据处理的**相关知识库**，支持通过自然语言查询 SQL 规则编写及数据流处理问题。用户可以提问，AI 会生成对应的 SQL 代码。例如：

- 我想检测设备「连续 3 次温度超过 100 度」的异常情况，应该如何结合窗口和过滤规则来实现？
- 如何将设备状态码（StatusCode） 0, 1, 2 分别映射为「停止，运行，故障」字符串输出？
- 温度传感器读数有抖动，我想保留小数点后 2 位，如何处理？

## 内置 Modbus TCP 模拟器



为了方便用户在**无硬件环境**下进行开发测试，3.8.0 版本内置了 **Modbus TCP 模拟器**。

**核心能力**

- 一键启动 Modbus TCP 模拟器，无需安装第三方软件。
- 点位可配置各类动态模拟数据。
- 一键导出模拟器配置，配置导入到南向设备，可自动完成数据采集功能。

**应用场景：**

- **开发测试**：在没有真实 PLC 的情况下，快速验证数据采集逻辑。
- **培训演示**：无需硬件设备，即可演示 NeuronEX 的完整功能。
- **CI/CD 集成**：在自动化测试流程中使用模拟器进行回归测试。

## 更多增强功能



除了上述亮点，3.8.0 版本还包含以下重要更新：

### 驱动能力增强



- **Modbus 驱动增强**：增加了配置点位时，对 Modbus 地址范围（0~65535）的强制校验，确保配置准确性，避免因地址错误导致的采集失败。
- **CODESYS V3 驱动增强**：Codesys 协议支持中文标签，方便中文用户使用。
- **OPC UA 驱动增强**：新增支持 **String Array（字符串数组）** 数据类型，进一步完善了 OPC UA 数据类型的覆盖范围。
- **OPC UA 驱动点位浏览功能**
  - **页面布局调整**：扩展 OPC UA 命名空间节点显示，更清晰地展示节点层次结构
  - **选中点位计数**：实时显示已选中的点位数量，方便批量操作
  - **批量导出与添加**：支持一次性导出或添加 10,000 个点位到采集组
- **驱动点位写入日志增强**：通过 MQTT 或 HTTP API 执行**点位写入操作**时，日志中新增以下信息，改进远程控制场景下的诊断效率。
  - **接口执行状态**：成功 / 失败
  - **报错详情**：如果失败，提供详细的错误原因（如地址范围错误、数据类型不匹配等）
  - **日志级别**：成功为 INFO 日志，失败为 ERROR 日志

### 其他优化项



- **RBAC 功能完善**：完善 Viewer 角色在数据分析、仪表盘及应用等页面的权限管控，确保数据安全。
- **规则调试功能**：规则调试功能的后端通信协议由 **WebSocket** 优化为 **SSE**，提升了调试的稳定性和实时性。
- **UI 风格调整**：调整并优化了 Dashboard 页面布局及 UI 风格。
- **内置 AI 功能**：新增支持 **Qwen 大模型**，为用户提供更多 AI 模型选择。
- **内置时序数据库**：内置时序数据库 Datalayers 版本更新为 **2.3.16**，性能和稳定性进一步提升。
