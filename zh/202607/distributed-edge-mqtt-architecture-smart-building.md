一家全球楼宇科技与工业自动化龙头企业，基于 EMQX Edge 搭建分布式边缘计算平台，汇聚全球数千栋建筑的传感器与设备数据。项目在多个园区部署轻量化 MQTT 消息服务器，形成可横向扩展的数据底座，支撑楼宇实时优化、预测性维护与 AI 智能分析等业务落地。

## 行业背景

现代楼宇管理系统早已从简单的暖通空调调度、能耗监测，升级为兼顾人员舒适度、运营效率与绿色低碳目标的智能化平台。智慧楼宇接入数百套感知设备（温度、人员情况、空气质量、设备遥测），与控制器、自动化系统与预测分析引擎协同运行。

随着楼宇智能化程度提升，传统的集中式云平台普遍面临网络时延高、带宽资源紧张、本地运行可靠性不足等痛点。行业因此转向分布式边缘计算架构：由本地边缘节点实时处理传感器数据，汇总后的统计数据再同步至中心云平台，用于跨园区数据分析与 AI 全局调优。

## 挑战：全球多园区规模化落地边缘智能

该企业旗下建筑资产遍布全球，业务对系统提出如下核心要求：

- **本地分布式边缘处理**

  写字楼、厂区、产业园区各自部署独立的消息服务器，空调调节、门禁联动、排水泵启停等指令在本地闭环执行，不依靠远距离云端通信。

- **海量终端接入能力**

  单个园区存在温控设备、PLC、照明控制器、边缘网关等数百台终端；人流峰值时段，全设备消息吞吐量达 80-100 万条 / 秒。

- **容器化原生部署**

  兼容 Docker 等容器环境，贴合企业现有运维体系与 CI/CD 持续交付流程。

- **可扩展数据持久化**

  对接 PostgreSQL 时序数据库，在园区本地留存设备运行记录，支撑历史数据查询、跨园区趋势研判。

- **就近园区互联互通**

  同一个城市内的多个园区边缘节点可以互相通信；后续所有站点统一接入集团总部云平台。

- **统一运维标准**

  前期选取 25 个园区作为试点，落地分布式边缘 MQTT 方案，后续计划扩容至全球 100+ 园区。

## 楼宇分布式自动化为何选用 MQTT 协议

MQTT 现已成为楼宇自动化、工业物联网通用标准，核心优势如下：

- **轻量化协议**

  很多建成多年的办公楼内网带宽有限，MQTT 采用精简二进制报文，极低的传输开销，即使成千上万的传感器同时上报数据，也不会造成网络拥堵。

- **发布/订阅模型天然适配楼宇场景**

  例如：暖通总控下发温度设定值变更指令，所有温控器、环境传感器自动订阅接收更新，天然匹配楼宇广播式控制场景。

- **分级可靠消息投递**

  消防报警、应急排风指令采用最高等级 QoS 服务质量等级，保障消息送达；温湿度等非核心遥测数据采用普通传输模式，在节省带宽的同时，保障安全设备优先运行。

- **边云无缝数据传输**

  园区 EMQX Edge 节点直接和总部云端桥接，不需要额外部署中间件，传感数据直接上传云端。

- **兼容多类工业协议**

  搭配 NeuronEX 边缘网关，把 Modbus、OPC‑UA 等老旧协议统一接入新一代自动化架构，设备无需硬件改造即可接入智能体系。

## EMQX Edge 智慧楼宇解决方案

EMQX Edge 是专为物联网边缘场景打造的高性能、超轻量化 MQTT 消息服务器，安装在园区本地工控机或者容器节点上，作为园区的数据中枢：接收楼层传感器、暖通控制器数据，就地完成逻辑判断，再把精简后的运行数据安全上传集团云平台或企业中心业务系统。 

该企业在所有园区站点部署 EMQX Edge，采用分布式边缘计算架构，具体部署模式如下：

### 部署模式

- **单园区独立部署**：每座写字楼或者厂区单独部署 EMQX Edge，负责本楼宇传感器采集、暖通和照明自动化策略执行。
- **分批次全球落地**：优先在核心办公园区投产验证方案，下一财年完成海外园区批量上线。
- **容器统一交付**：依靠标准化 Docker 镜像，不管是国内机房环境，还是海外不同厂商服务器环境，运维人员可以快速完成部署和扩容。

### 单园区承载能力

- **多类型设备统一接入：**一台 EMQX Edge 同时接入环境传感器、PLC、网关、楼宇控制器等 OT 设备，设备种类繁多也不会出现连接拥挤。
- **峰值场景稳定运行：**上下班人员流动、节假日设备启停等高峰期，百万级消息平稳处理，安全告警、设备调节指令毫秒级响应。
- **报文适配多样化设备：**既可以接收传感器简短状态报文，也能接收控制器大容量状态文件，适配高密度传感设备数据上报。
- **测试环境独立隔离：**搭建单独预发布环境，新版本程序、固件升级、应急演练都在测试节点完成，不会干扰园区正在运行的设备。

### 数据集成架构

EMQX Edge 通过 PostgreSQL 数据桥接持久化存储园区汇总数据，同时与云端 EMQX 建立实时数据桥接通道，实现数据实时同步。

![image.png](https://assets.emqx.com/images/1f7d6f69c01e27768d685ce1111cf5ac.png)

 该架构实现三类核心能力：

1. **本地实时自动化**：内置 MQTT 规则引擎解析传感器数据，毫秒级下发指令驱动本地控制器执行动作。
2. **时序数据本地存储**：PostgreSQL 缓存园区全量遥测数据，用于本地数据分析、趋势展示与合规审计溯源。
3. **轻量化云端同步**：仅将每小时园区汇总指标通过 EMQX 同步至总部系统，供跨园区 AI / 机器学习建模分析，避免原始数据流占用大量带宽。

## EMQX 核心能力

- **轻量化 MQTT 服务**：资源占用极低，适配园区容器化部署，完整兼容 MQTT 5.0 协议标准。
- **高并发消息处理**：单实例每秒处理 80 万‑100 万条消息，从容应对人流突变、应急事件等数据高峰。
- **PostgreSQL 数据持久桥接**：网络临时断开时本地缓存数据，网络恢复之后自动把设备运行记录补传到数据库。
- **容器化部署**：统一 Docker 镜像与配置模板，各类园区 IT 环境可快速上线，大幅减少现场调试工作量。
- **持久化会话管理**：网络短暂波动之后，传感器和控制器自动恢复连接，不用人工逐个重启设备。
- **可视化规则引擎**：运维人员在边缘侧直接配置阈值告警、状态联动暖通调控等自动化策略，消除云端往返时延
- **边缘集群互联预留能力**：支持邻近园区边缘节点互通桥接，后续平滑接入集团云端平台。

## 业务落地价值

部署 EMQX Edge 后，企业楼宇管理体系升级为高可靠、低成本分布式数据网络：

- **企业规模化快速拓展**：试点园区验证方案可行后，新建园区部署周期由数月缩短到几周，降低海外项目落地难度。

- **为后期智能化升级铺路：**统一 MQTT 标准接入全品类环境传感器、PLC、控制器，为混合云 - 边缘联合分析、跨园区预测性维护打好基础。
