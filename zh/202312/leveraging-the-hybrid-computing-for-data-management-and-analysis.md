在[上一篇文章](https://www.emqx.com/zh/blog/practical-data-management-for-smart-manufacturing)中，我们对智能制造中的数据管理过程进行了全面的探索，介绍了数据采集、数据映射、数据标准化和数据情景化等重要阶段，明确了在数据管理这个充满活力的领域中要管理“什么”。在本篇文章中，我们将重点讨论数据管理流程中同样重要的“如何”管理和“在哪里”管理。

在复杂的数据管理环境中，边缘计算和云计算有着巨大的差异，是两种完全不同的计算模式。边缘计算使得实时处理更加贴近数据源，而云计算则为高级数据分析提供了强有力的支持。现在让我们设想一个新世界，在这里无需在这两种计算能力之间做出艰难选择，而是可以充分发挥它们的综合优势——这就是混合计算的，一个将边缘计算和云计算相融合的引人注目的领域，为您带来更全面、灵活的计算体验。

本文将深入探讨混合计算模型的变革潜力，这种模型巧妙地融合了边缘计算和云计算的强大能力，为用户提供了全方位的数据管理和分析解决方案。我们将通过一个实际案例展示 [EMQX](https://www.emqx.com/zh/products/emqx)、[Neuron](https://neugates.io/zh) 和 [eKuiper](https://ekuiper.org/zh) 如何协同合作，以优化边缘和云计算环境中的数据利用，从而为用户提供更高效的计算体验。

## 通过混合模型实现数据 ETL

为了说明混合模型的强大威力，让我们继续之前 EMQX 和 Neuron 的示例。这一次，我们还引入了边缘流处理引擎 eKuiper，以提供边缘处理能力。在云端，所有清洗后的数据都被发布到 EMQX Broker，形成一个统一命名空间。各种应用（如 Kafka）可以订阅相关的主题以进行流处理。这些组件共同呈现了边缘计算和云计算之间的协调互动。从实时数据采集到基于云的分析，该真实示例生动展示了我们解决方案的潜力。

就像前一篇文章中所介绍的那样，[Neuron](https://neugates.io/zh) 是一个主设备，连接多个从设备，包括 3 个电力表和 1 个温度读取器。每个从设备都有一个设备 ID 用于标识数据源。Neuron 通过轮询这些从设备，逐一获取当前的读数。Neuron 成为第一个执行数据映射的环节，将 [Modbus](https://www.emqx.com/zh/blog/modbus-protocol-the-grandfather-of-iot-communication) 从设备读取的数据与我们所关注的真实世界模型对象——化学罐进行映射。Neuron 能够从各种数据源中提取数据，这是数据提取过程。

化学罐的数据随后被发送至 [eKuiper](https://ekuiper.org/zh) 进行数据清洗和转换。eKuiper 具备管理多种数据源的能力，可以通过 API 或其他通信协议访问 MES 和 ERP 的生产数据。eKuiper 对来自 Neuron 的化学罐对象数据进行标准化，并向化学罐添加情景数据。这是数据转换过程。

通过 [MQTT](https://www.emqx.com/zh/blog/the-easiest-guide-to-getting-started-with-mqtt) 通信协议，将经过情景化的数据发布到 [EMQX](https://www.emqx.com/zh/products/emqx) Broker。EMQX 将充当中央数据存储库，汇总来自各个地点的所有数据。所有化学罐对象的数据都将按照 ISA95 标准层次结构进行组织。这是数据加载的过程。

上述过程构成了数据仓库的完整画面，其中提取、转换和加载（ETL）是将数据从多个源整合到一个大型中央存储库的过程。EMQX、Neuron 和 eKuiper 共同为第三方应用软件（如 Kafka）实现了统一命名空间功能。

EMQX 内置了强大的规则引擎，能够将化学对象的数据导入 Kafka 流式平台。三个化学罐的数据流被储存在相应的 Kafka 主题中，用于进行数据分析，包括机器学习、深度学习或更为复杂的计算模型。

总的来说，边缘计算和云计算在混合模式下完美融合，借助 EMQX、Neuron 和 eKuiper 的支持，提供了一套全面的数据管理解决方案。这种混合模型充分考虑了两者的优势，实现了实时洞察和快速行动的同时，还能充分利用先进的分析和可扩展的存储，为数据管理提供多维度、全方位的方法。在这个数据驱动的世界中，这种协同效应已成为颠覆性的因素，促进了更智能的决策并提高了运营效率。

## 混合模式的优势

每一种计算模式都有其独特的优势。边缘计算有助于数据的高效管理，而云计算则为数据分析提供了强大的支持。当这两种模式实现无缝融合时，便能够实现协同效益，尤其是在智能制造领域。

### 边缘计算的优势

1. **低延迟：**边缘计算在数据产生的地方进行本地处理，与远程云服务器相比大大减少了数据处理延迟。在智能制造中，低延迟对于实时决策以及控制机器和过程非常重要。
2. **实时数据处理：**边缘设备具备实时处理数据的能力，能够迅速响应工厂现场的变化。这在制造业中是不可或缺的，对于质量控制、预测性维护和过程优化都至关重要。
3. **带宽效率：**通过在边缘进行数据处理，仅将相关信息传送至云端，有效减少了网络传输的数据量。这在网络带宽有限或昂贵的情况下尤为有益。
4. **提高安全性：**边缘计算通过将敏感数据存储在制造设施内，提升了数据安全性，减少了数据泄露和未经授权访问的风险。利用边缘计算，能够更精准地管理数据的实际存储位置和访问权限。
5. **冗余和弹性：**边缘计算具备冗余和故障转移的功能。即使某个边缘设备发生故障，制造过程仍可继续进行，尽量减少中断。相比之下，云数据中心可能面临停机的风险。
6. **节省成本：**边缘计算有助于降低云端传输和存储数据的成本，因为只有有价值的数据才会被发送到云端进行长期存储和分析。这能够持续节约成本。

### 云计算的优势

与边缘计算相比，云计算在智能制造的数据分析方面具有一定优势。

1. **数据集中管理：**把数据集中在云端，可以让多个制造工厂的数据在同一地点进行存储、处理和分析。这能够促进跨工厂的协调、基准测试和全面洞察。
2. **高级分析：**云计算平台提供了强大的分析和机器学习工具，可实现大规模数据的高效处理与分析。这使得制造商能够获得更深入的洞察，进行预测性维护，更有效地优化生产流程。
3. **协作和远程访问：**基于云的系统支持协作和远程访问数据以及分析工具，这在专家或相关方需要从不同地点访问和共享制造数据时非常有用。
4. **先进技术的可及性：**云服务提供商不断投入研发，使得制造商更容易接触和利用前沿技术，如人工智能、物联网和大数据分析。
5. **与第三方服务的集成：**云服务可以轻松地与第三方工具和服务集成，为制造商提供更多的解决方案选择，包括外部数据源和行业专用软件。
6. **全球数据处理和洞察：**云计算使得制造商能够同时处理和分析来自全球多个地点的数据，有助于全方位了解运营和供应链的状况。

## 结语

值得强调的是，对于智能制造而言，混合模式往往是最实用的解决方案。在这种模式下，边缘设备负责实时的数据收集和处理，而云端则用于长期数据存储、高级分析和全球协调。这种方法充分发挥了边缘计算和云计算的各自优势，从而优化了制造业的运营。在边缘计算和云计算之间进行选择时，应基于具体制造环境的需求和限制做出决策。
